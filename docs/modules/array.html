binarySearch

  • binarySearch<T, K>(searchValue: K | K[], sort?: SortFn, property?: string | number): OperatorFunction<T | T[], BinarySearchResult>
    • Defined in libs/rxjs/array/src/lib/binary-search.ts:79

    The binarySearch operator takes an Observable array of values T and returns a BinarySearchResult containing the sorted array, search value and index of the search value.

    The default sorting method for this search is a basic quality check, if you need more complex sorting such as objects you can pass a sorting method and additional property to search on

    remarks

    When using an additional property, if it's a number the underlying T[] is assumed to be an array. If you have an object with a number property, use a string value instead (e.g. '5' instead of 5)

    example
    of([1, 4, 7, 2, 5, 6, 3, 8, 10, 9])
 .pipe(binarySearch(5), take(1))
 .subscribe(console.log) // [4, 5, [1, 2, 3,...]]
    example
    const sort = (a: { val: number }, b: { val: number }) => {
 if (a.val === b.val) return 0;
 return a.val < b.val ? -1 : 1;
};

of([
 { val: 1 }, { val: 4 }, { val: 7 }, { val: 2 }, { val: 5 },
 { val: 6 }, { val: 3 }, { val: 8 }, { val: 10 }, { val: 9}
])
 .pipe(binarySearch<{ val: number }, number>(5, sort, 'val'), take(1))
 .subscribe(console.log)
 // [4, 5, [{ val: 1 }, { val: 2}, { val: 3 },...]]
    example
    from([1, 4, 7, 2, 5, 6, 3, 8, 10, 9])
 .pipe(binarySearch(5), take(1))
 .subscribe(console.log) // [4, 5, [1, 2, 3,...]]
    example
    const sort = (a: [number, number], b: [number, number]) => {
 if (a[1] === b[1]) return 0;
 return a[1] < b[1] ? -1 : 1;
};

from<[number, number][]>([
 [1, 1], [2, 4], [3, 7], [4, 2], [5, 5],
 [6, 6], [7, 3], [8, 8], [9, 10], [10, 9]
])
 .pipe(binarySearch<[number, number], number>(5, sort, 1), take(1))
 .subscribe(console.log)
 // [4, 5, [[1, 1], [4, 2], [7, 3],...]]

    Type parameters

    • T

      The type in the array to search over

    • K

      The type of the value in array item to search

    Parameters

    • searchValue: K | K[]

      The value to search for in the source array

    • Optional sort: SortFn

      Optional sort method for sorting more complex types

    • Optional property: string | number

      Optional property to be searched on, use a number for array index and strings for object keys

    Returns OperatorFunction<T | T[], BinarySearchResult>

difference

  • difference<T>(input: T[] | ObservableInput<T[]>, inputModifier?: MapFn<T, T>): MonoTypeOperatorFunction<T[]>
    • Defined in libs/rxjs/array/src/lib/difference.ts:53

    The difference can be used with an Observable array of values of T, passing in an array or Observable array of values to find the difference between the two. The returned array of values only contains values from the source Observable

    An optional method can be used to convert values for comparison (see examples)

    example
    of(['a', 'b', 'd'])
 .pipe(difference(['a', 'c']))
 .subscribe(console.log) // ['b', 'd']
    example
    of(['a', 'b', 'd'])
 .pipe(difference(of(['a', 'c'])))
 .subscribe(console.log) // ['b', 'd']
    example
    of(['a', 'b', 'd'])
 .pipe(difference(['A', 'C'], (value) => value.toUpperCase()))
 .subscribe(console.log) // ['b', 'd']
    example
    of(['a', 'b', 'd'])
 .pipe(difference(of(['A', 'C'], , (value) => value.toUpperCase())))
 .subscribe(console.log) // ['b', 'd']

    Type parameters

    • T

      The type of data in the input array

    Parameters

    • input: T[] | ObservableInput<T[]>

      An Observable Array of items use to get the difference between two arrays

    • Optional inputModifier: MapFn<T, T>

      A function that provide modification of the values to do the comparision with

    Returns MonoTypeOperatorFunction<T[]>

    Array of values of difference between the source and input array

sort

  • sort<T>(fn?: SortFn): MonoTypeOperatorFunction<T[]>
    • Defined in libs/rxjs/array/src/lib/sort.ts:43

    The sort operator takes an array of T items and returns the array sorted.

    By default the operator can do basic quality check on string, number and boolean values, and can take an optional method to do more complex searches for items such as array or objects

    example
    of([2, 4, 6, 1, 3, 5])
 .pipe(sort(), tap(console.log))
 .subscribe() [1, 2, 3, 4, 5, 6]
    example
    
function sortTuple(a, b) {
   if (a[1] === b[1]) return 0;
   return a[1] < b[1] ? -1 : 1;
}

of([ [10, 2], [20, 4], [30, 6], [40, 1], [50, 3], [60, 5] ])
.pipe(sort(sortTuple), tap(console.log))
.subscribe() // [ [40, 1], [10, 2], [50, 3], [20, 4], [60, 5], [30, 6] ]

    Type parameters

    • T: unknown

      The type of data in the input array

    Parameters

    • Optional fn: SortFn

      Optional sorting function

    Returns MonoTypeOperatorFunction<T[]>

    Array of sorted values
